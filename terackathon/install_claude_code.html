1. VS Codeのインストールおよび設定

すべての作業のベースとなるエディタを導入し、最低限の開発設定を行います。

  1. Visual Studio Code公式サイトにアクセスする。
  2. OS（Windows/Mac）に合ったインストーラーをダウンロードする。
  3. 画面の指示に従ってインストールを完了させる。
  4. VS Code左端のアクティビティバーから「拡張機能（四角いブロックのアイコン）」をクリックする。
  5. 検索バーに Cline と入力し、インストールする。
  6. C/C++やPythonなど、自身の開発用途に合わせた拡張機能も併せてインストールしておく。

2. Claude Codeのインストールおよび設定

現在、最も推奨される方法は、依存関係が少なく自動アップデートにも対応したネイティブインストーラーです。

  1. VS Codeを開き、上部メニューの「ターミナル」>「新しいターミナル」で画面下部にターミナルを開く。
  2. 使用中のOS・シェルに合わせて、以下のコマンドを実行する。

macOS / Linux（WSL含む）

curl -fsSL https://claude.ai/install.sh | bash

Windows（PowerShell・推奨）

irm https://claude.ai/install.ps1 | iex

Windows（CMD）

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
  1. インストール完了後、作業したいプロジェクトのディレクトリへ移動し、claude を実行する。
  2. 初回起動時にブラウザが開くので、Anthropicアカウントでログイン（認証）を行う。
  3. 認証完了後、ターミナル上で自然言語によるコーディング指示が可能になる。
  4. 必要に応じて claude --version を実行し、インストール状態を確認する。
補足1: 以前よく使われていた npm install -g @anthropic-ai/claude-code は、現在は非推奨（レガシー）扱い。
補足2: 利用には Claude の有料プラン（Pro/Max/Teams/Enterprise）または Anthropic Console のAPIクレジットが必要。
補足3: Windowsネイティブ環境では、Bash系ツールを扱いやすくするために Git for Windows の導入が推奨される（WSL環境では不要）。

3. Gitの概念説明、GitHubアカウント取得、Gitインストール

コマンドを覚える前に、まずはGitの基本概念と運用ルールを押さえたうえで、アカウントと実行環境を準備します。

Gitってなに？

Gitは、ファイルの変更履歴を管理する仕組みです。
「いつ」「誰が」「どこを」「なぜ変えたか」を記録し、必要なら過去の状態に戻したり、変更差分を比較したりできます。

ローカルとリモート

  • ローカル: 自分のPC内にある作業コピー
  • リモート: GitHub上にある共有コピー

基本の流れ（重要）

  1. ファイルを編集する（ローカル）
  2. git add で変更を記録対象にする
  3. git commit で履歴として確定する（ここまでローカル）
  4. git push でリモート（GitHub）に反映する
補足: リモートへ反映するのは push。pull はリモートの変更をローカルへ取り込む操作。

ブランチ運用（自分だけの世界線）

  • main（または master）は、常に動く安定版として扱う
  • 機能追加や編集は、必ず main 以外のブランチで行う
  • 直接 main にコミットせず、作業ブランチでコミットしてからPRで統合する

基本手順:

  1. main から作業ブランチを作る（例: feature/login）
  2. 作業ブランチで編集・コミットする
  3. GitHubでPull Requestを作る
  4. レビュー後に main へマージする

GitHubアカウントの取得

  1. GitHub公式サイトにアクセスする。
  2. 右上の「Sign up」をクリックする。
  3. メールアドレス、パスワード、ユーザー名を入力し、アカウントを作成する。

Gitのインストール

  • Windows: Git for Windows をインストール
  • Mac: Homebrewを使う場合は brew install git

インストール後、ターミナルで次を実行して確認する。

git --version

4. VS Code・Claude Code・Gitの連携設定

3つをつないで、AIがGit運用を補助できる状態にします。

  • CLI（Claude Code）でのGit操作
    ネイティブインストーラーで導入したClaude Codeは、フォルダがGit管理（git init）されていればGitコマンドを使ってコミット支援できます。
    例: 「今の変更をコミットして」と指示すると、git add と git commit を提案してくれます。
  • GUI（Cline）でのMCP活用
    ClineはMCP（Model Context Protocol）をサポートしており、外部ツールとの連携が得意です。
    Clineのサイドバー設定からMCPサーバーを追加すれば、GitHub Issue管理などの周辺タスクにも展開できます。

5. 総括した開発フロー（VS Codeでの実践）

例として、M5StackやESP32から送られてくるセンサーデータを処理する、簡単なPythonサーバーを開発する流れを示します。

  1. プロジェクトの準備
    PC上に空のフォルダ（例: sensor-logger）を作成し、VS Codeでそのフォルダを開く。
  2. Claude Codeの起動
    VS Codeのターミナルで claude を実行する。
  3. 要件を伝える（指示出し）
    Pythonでセンサーデータを受信して保存する簡単なアプリを作りたい。ファイルの構成案を出し、必要なライブラリのインストールとベースとなるコードを作成して。
  4. 確認と実行
    Claude Codeがファイル作成や依存ライブラリの追加コマンドを提案したら、内容を確認して承認する。
  5. ブランチを切って開発する
    main から作業ブランチを作り、そのブランチ上で実装する。
  6. Gitで履歴管理する
    変更を add → commit し、区切りごとに push する。
  7. PRでmainへ統合する
    GitHubでPull Requestを作成し、レビュー後に main へマージする。

VS Codeという一つのウィンドウ内で、「コード編集」「AI支援」「Git運用」を一体化して進めるのが、現代の効率的な開発スタイルです。
